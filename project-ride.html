Scheduled-Ride Feature

A personal project scoping and designing a scheduled-ride feature for ride-hailing apps — solving the unreliable last-minute booking problem for time-sensitive trips like airport runs and early morning commutes.

Project Type
Personal Project
Industry
Mobility / Tech
Tools Used
Balsamiq, Research

The Problem

Ride-hailing apps like Bolt and Uber work well for spontaneous trips. But for planned, time-sensitive journeys — early morning flights, important meetings, hospital appointments — the uncertainty of finding a driver last-minute creates real anxiety for users.

My Approach

This project started with a personal frustration and turned into a structured product discovery:

  • Mapped out the user journey for time-sensitive trips
  • Identified the key pain points — driver availability, surge pricing, uncertainty
  • Researched how existing apps handle scheduling and what gaps remain
  • Began wireframing the scheduling flow using Balsamiq

Key Features Being Designed

  • Schedule a ride up to 7 days in advance
  • Driver confirmed at least 1 hour before pickup
  • Price locked at booking — no surge at the time of the ride
  • Reminder notifications for both driver and passenger
  • Easy rescheduling and cancellation with clear policies

Current Status

This project is still in progress. The problem definition and feature set are largely complete. Wireframes are currently being built in Balsamiq and will be updated here once ready.
